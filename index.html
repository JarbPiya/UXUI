Design System / Coffee Shop Admin / Premium SaaS

Premium coffee operations

A calm design system for specialty coffee teams.

This single HTML file documents the visual language, tokens, component states, reusable patterns, and admin screens needed for a modern coffee shop management platform.

System principle

Quiet controls, clear hierarchy.

Use warm surfaces, restrained accents, soft borders, and direct content. Avoid decoration that does not support operations.

Team Handoff

Shared rules make this easier for designers and engineers to extend without breaking consistency.

Atomic namingFoundation → Component → Pattern → Screen

Example: Component/Button/Primary and Pattern/POS/Product Card.

Component rulesOne component, explicit variants.

Avoid duplicate card, badge, button, and table styles. Extend with modifier classes.

AccessibilityUse semantic HTML first.

Buttons are buttons, tables are tables, focus states are visible, and labels are attached to inputs.
